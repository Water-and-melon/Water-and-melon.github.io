ThreadPoolExecutor源码浅析

发表于 2017-02-16 | 分类于 java

前言

最近在看JUC中的线程池，分析下ThreadPoolExecutor。

总体逻辑

线程池中存在一些工作线程和一个任务队列，工作线程是可以复用的，完成一个任务后可以继续完成下一个任务，工作线程的数量是有上限的，如果任务数量超过了工作线程的数量的上限，就会把任务放在一个任务队列中等待。

源码浅析

Worker

worker是工作线程类，对线程进行了一定的封装。

发现worker居然继承AbstractQueuedSynchronizer，可以用worker对象进行一些同步的操作，重写的函数较为简单，简单的互斥锁，1表示锁定，0表示没有锁定。

ThreadPoolExecutor

ThreadPoolExecutor就表述一个完整的线程池，首先了解下一些关键的属性。

需要关注的是5个线程池的状态：

  • RUNNING：
  • SHUTDOWN：此时线程池不能够接受新的任务，它会等待所有任务执行完毕。
  • STOP：此时线程池不能接受新的任务，并且会去尝试终止正在执行的任务
  • TIDYING：所有工作线程已经销毁，任务缓存队列已经清空
  • TERMINATED：线程池彻底终止

corePoolSize和maximumPoolSize的不同：

  • corePoolSize是核心线程数量，表示理想状态下的线程数量
  • maximumPoolSize是最大线程数量，当任务数量超过理想值时，不得不新增线程数量，但是新增的线程数量也是有上限的。如果当前的线程数量大于corePoolSize且当前的任务数量少于当前线程数量。如果小于则不回收，除非开启allowCoreThreadTimeOut。
