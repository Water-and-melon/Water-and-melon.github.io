设计模式：变继承为组合

发表于 2017-03-07 | 分类于 设计模式 |

前言

变继承为组合绝对是设计模式的最为精髓的地方，很多的设计模式都是追求的这一点，本文通过一些例子谈谈我自己的对这些的认识。

装饰者模式就是去组合，把自己的方法和装饰的内容进行组合，组合后的东西可以再次去组合。这样就非常灵活，耦合度非常低。就好比你穿衣服，可以有各种组合。

另外的设计模式

比如策略模式，策略模式如果用继承来实现，就是定义一个策略基类，然后继承来实现具体的策略，客户端再去调用，这里就有很多的冗余，各个类除了策略的不同，其他的东西都是一样的。我们要做的只需要把这个策略组合进行，这样，除了策略其他的代码就不存在冗余。

总结

变继承为组合的设计模式还有很多，如状态模式、代理模式、适配器模式等，理解变继承为组合就可以更好的理解很多设计模式
